# 会话摘要更新任务

请将下面的“已有摘要”和“新增对话”合并，生成一份更新后的摘要，用于后续继续问答。

## 更新目标

更新后的摘要应尽量保留以下内容：

- 用户持续关注的核心问题
- 已经讨论过的重要模块、算法、参数、接口、页面或流程
- 已经明确的关键结论
- 当前仍未解决或仍值得继续追问的问题

## 更新要求

1. 新摘要应在已有摘要基础上做整合与压缩，而不是简单拼接。
2. 重复内容应合并，过于细碎但无后续价值的信息可以省略。
3. 不要加入原始内容中没有的新事实。
4. 如果新增对话中出现了类似 `[用户反馈：...]` 的附加标记，应该将其视为用户的情绪价值并参考给后续回答调优，不要写入摘要。
5. 在尽量保证信息完整的前提下，将长度控制在 `{SUMMARY_TARGET_CHARS}` 字左右，不要超出太多。
6. 输出必须使用 Markdown。
7. 不要输出额外说明，不要写“新摘要如下”。

## 建议输出结构

### 用户关注点

- ...

### 已确认信息

- ...

### 未解决问题

- ...

{prev_summary}
