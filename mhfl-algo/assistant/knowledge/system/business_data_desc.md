# 业务数据说明

本段内容用于说明后续业务数据 JSON 的含义与使用方式。

业务数据属于平台的动态上下文，主要用于回答“当前有什么”“当前是多少”“当前状态如何”这一类问题。  
当问题涉及平台当前任务、当前算法列表、当前数据集列表、用户自己的任务情况时，应优先参考业务数据。  
当问题涉及概念解释、算法原理、字段含义、训练流程、接口机制、系统架构时，应优先参考知识库，而不是仅依据业务数据。

## 可出现的业务数据类型

### 任务数据

任务数据通常用于回答以下问题：

- 我的任务有哪些
- 最近任务有哪些
- 当前任务总量是多少
- 某个任务的状态是什么
- 某个任务使用了什么算法或数据集
- 某个任务的训练参数是什么

任务数据中可能包含但不限于以下信息：

- 任务 `id`
- 数据集名
- 算法名
- 任务状态
- 创建时间
- 更新时间
- 训练参数
- 训练指标

### 算法数据

算法数据通常用于回答以下问题：

- 平台有哪些算法
- 当前支持哪些联邦学习算法
- 是否支持某个算法
- 算法列表有哪些

算法数据中可能包含但不限于以下信息：

- 算法 `id`
- 算法名

### 数据集数据

数据集数据通常用于回答以下问题：

- 平台有哪些数据集
- 当前支持哪些数据集
- 是否支持某个数据集
- 数据集列表有哪些

数据集数据中可能包含但不限于以下信息：

- 数据集 `id`
- 数据集名

## 使用原则

1. 如果问题是在询问平台当前状态、当前数量、当前列表、当前任务情况，应优先依据业务数据回答。
2. 如果业务数据中没有提供某项信息，应明确说明当前业务数据不足，不能自行补造。
3. 不要把静态知识解释成实时业务数据。
4. 不要把通用常识写成平台当前事实。
5. 如果问题同时涉及“当前情况”和“平台规则”，应在回答中清楚区分两者。
