# 会话摘要任务

请根据下面的对话内容，整理出一份可供后续问答继续使用的摘要。

## 摘要目标

摘要应尽量保留以下内容：

- 用户当前关注的核心问题
- 已经讨论过的重要模块、算法、参数、接口、页面或流程
- 已经明确的关键结论
- 仍未解决或仍需继续追问的问题

## 摘要要求

1. 摘要应简洁、准确、连贯，并适合后续继续作为上下文使用。
2. 不要大段照抄原始对话。
3. 不要加入对话中没有出现的新事实。
4. 如果对话中出现了类似 `[用户反馈：...]` 的附加标记，应该将其视为用户的情绪价值并参考给后续回答调优，但是不要写入摘要。
5. 在尽量保证信息完整的前提下，将长度控制在 `{SUMMARY_TARGET_CHARS}` 字左右，不要超出太多。
6. 输出必须使用 Markdown。
7. 不要输出额外说明，不要写“摘要如下”。

## 建议输出结构

### 用户关注点

- ...

### 已确认信息

- ...

### 未解决问题

- ...

对话记录：
{hist}
