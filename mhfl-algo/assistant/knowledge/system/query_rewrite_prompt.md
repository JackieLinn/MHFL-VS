# 查询改写任务

请针对下面的用户问题，生成 {n} 个不同表述的改写问题，用于检索相关文档。

## 改写目标

改写后的问题应尽量覆盖同一意图下的不同表达方式，以提升知识库召回效果。  
改写时可以补全关键词、替换更常见的表述、拆解较模糊的问题，但不能偏离原问题的核心意图。

## 改写要求

1. 保持原始问题的核心含义不变。
2. 优先保留原问题中的算法名、数据集名、接口名、参数名、字段名、状态名、模块名。
3. 若原问题表达较口语化，可以改写为更适合文档检索的表达。
4. 若原问题较宽泛，可以改写为更具体、更有检索价值的问题。
5. 不要引入原问题中没有依据的新事实。
6. 不要把完全不同的问题混入改写结果。
7. 不要回答问题，只做改写。

## 输出要求

1. 严格输出 {n} 行内容。
2. 每行只写一个改写问题。
3. 不要编号。
4. 不要使用项目符号，例如 `-`、`*`、`•`。
5. 不要输出标题、说明、引号、前缀或任何额外文字。

用户问题：{question}
